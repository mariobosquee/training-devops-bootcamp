Instalacion

Instalar ArgoCD en el cluster

# Crear namespace dedicado
kubectl create namespace argocd

# Instalar ArgoCD desde el manifiesto oficial
kubectl apply -n argocd \
  -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

# Verificar que los pods estan corriendo
kubectl get pods -n argocd
# Debeis ver 5-7 pods en estado Running

Acceso a la UI

Acceder al dashboard web

Obtener la contrasena del admin:

kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath="{.data.password}" | base64 -d

Exponer el servidor con port-forward:

kubectl port-forward svc/argocd-server -n argocd 8443:443

Abrir en el navegador: https://localhost:8443

  • Usuario: admin
  • Contrasena: la obtenida con el comando anterior
Nota

El navegador mostrara un aviso de certificado. Aceptadlo con Advanced > Proceed.

CLI

Comandos esenciales del CLI

# Iniciar sesion contra el servidor
argocd login localhost:8443 --insecure

# Listar todas las aplicaciones
argocd app list

# Ver detalle de una aplicacion
argocd app get mi-app

# Sincronizar (desplegar) una aplicacion
argocd app sync mi-app

# Eliminar una aplicacion
argocd app delete mi-app

# Ver diferencias entre Git y cluster
argocd app diff mi-app

Application via UI

Crear Application desde la UI

  • 1 Click en NEW APP en el dashboard
  • 2 Rellenar Application Name y seleccionar Project: default
  • 3 Elegir Sync Policy: Manual o Automatic
  • 4 En Repository URL pegar la URL del repo Git
  • 5 Indicar el Path dentro del repo donde estan los manifiestos (ej: k8s/)
  • 6 En Cluster URL usar https://kubernetes.default.svc y elegir el Namespace
  • 7 Click en CREATE y luego SYNC para desplegar

Application via YAML

Crear Application con manifiesto

yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: mi-app
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/usuario/repo.git
    targetRevision: HEAD
    path: k8s
  destination:
    server: https://kubernetes.default.svc
    namespace: default
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
kubectl apply -f application.yaml

Sync y Rollback

Sincronizar y revertir

# Sincronizar la aplicacion con el estado de Git
argocd app sync mi-app

# Forzar sync (elimina recursos huerfanos y recrea)
argocd app sync mi-app --force

# Ver historial de despliegues
argocd app history mi-app

# Rollback a una revision anterior (ej: revision 2)
argocd app rollback mi-app 2
Importante

El rollback revierte el cluster a un estado anterior, pero no cambia Git. Si el auto-sync esta activo, ArgoCD volvera a sincronizar desde Git. Para un rollback permanente, revertid el commit en el repositorio.

Flujo tipico

Workflow GitOps con ArgoCD

  • 1 Desarrollador hace cambios en los manifiestos YAML del repositorio Git
  • 2 Se crea un Pull Request y se revisa el cambio
  • 3 Al hacer merge a main, Git queda actualizado con el estado deseado
  • 4 ArgoCD detecta la diferencia entre Git y el cluster (OutOfSync)
  • 5 ArgoCD sincroniza automaticamente (o manualmente con click en SYNC)
  • 6 El cluster refleja el estado deseado — la app aparece como Synced
  • 7 Si algo falla, revertir el commit en Git y ArgoCD aplica el rollback
