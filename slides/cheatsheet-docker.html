Docker

Referencia rapida

IMAGENES

Imagenes

bash
docker pull nginx:alpine # Descargar imagen de Docker Hub docker build -t mi-app . # Construir imagen desde Dockerfile docker images # Listar imagenes locales docker rmi nginx:alpine # Eliminar una imagen docker tag mi-app user/mi-app:v1 # Etiquetar imagen para registry docker push user/mi-app:v1 # Subir imagen a Docker Hub
CONTENEDORES

Contenedores

bash
docker run nginx:alpine # Crear y arrancar contenedor docker ps # Listar contenedores activos docker ps -a # Listar todos (incluidos parados) docker stop mi-contenedor # Parar un contenedor docker start mi-contenedor # Arrancar un contenedor parado docker rm mi-contenedor # Eliminar un contenedor docker logs mi-contenedor # Ver logs del contenedor docker logs -f mi-contenedor # Seguir logs en tiempo real docker exec -it mi-contenedor sh # Abrir terminal dentro del contenedor
FLAGS DE RUN

Flags importantes de docker run

bash
-d # Ejecutar en segundo plano (detached) -p 8080:80 # Mapear puerto host:contenedor -v ./datos:/app/data # Montar volumen host:contenedor --name mi-app # Asignar nombre al contenedor -e NODE_ENV=production # Definir variable de entorno --rm # Eliminar contenedor al parar

Ejemplo combinado

bash
docker run -d \ --name api \ -p 3001:3001 \ -v ./data:/app/data \ -e NODE_ENV=production \ --rm \ mi-app:latest
DOCKERFILE

Dockerfile

Dockerfile
FROM node:18-alpine # Imagen base WORKDIR /app # Directorio de trabajo COPY package*.json ./ # Copiar archivos de dependencias RUN npm install # Ejecutar comando en build COPY . . # Copiar el resto del codigo ENV NODE_ENV=production # Variable de entorno EXPOSE 3001 # Documentar puerto (informativo) CMD ["node", "src/index.js"] # Comando de arranque
Buena practica: copiar primero package*.json e instalar dependencias antes de copiar el codigo. Asi Docker cachea la capa de npm install y solo la reconstruye si cambian las dependencias.
DOCKER COMPOSE

Docker Compose

bash
docker compose up -d # Levantar servicios en segundo plano docker compose down # Parar y eliminar servicios docker compose ps # Ver estado de los servicios docker compose logs # Ver logs de todos los servicios docker compose logs api # Ver logs de un servicio concreto docker compose build # Reconstruir imagenes docker compose exec api sh # Terminal dentro de un servicio

Ejemplo docker-compose.yml

docker-compose.yml
services: api: build: ./backend ports: - "3001:3001" environment: - NODE_ENV=production volumes: - ./data:/app/data frontend: build: ./frontend ports: - "3000:80" depends_on: - api
LIMPIEZA

Limpieza

bash
docker system prune # Eliminar todo lo no usado (contenedores, redes, imagenes dangling) docker system prune -a # Eliminar TODO lo no usado (incluye imagenes sin contenedor) docker image prune # Eliminar imagenes dangling docker image prune -a # Eliminar todas las imagenes sin uso docker volume prune # Eliminar volumenes no usados
Usa docker system prune regularmente para liberar espacio en disco. Docker acumula imagenes y contenedores parados rapidamente.
FLUJO TIPICO

Flujo tipico de trabajo

  • 1 Escribir el Dockerfile — Definir la receta de tu aplicacion
  • 2 Construir la imagen — docker build -t mi-app .
  • 3 Ejecutar el contenedor — docker run -d -p 3001:3001 mi-app
  • 4 Verificar que funciona — docker ps y docker logs
  • 5 Iterar — Cambiar codigo, reconstruir imagen, recrear contenedor
  • 6 Compose para multi-servicio — docker compose up -d
  • 7 Publicar — docker tag + docker push al registry
