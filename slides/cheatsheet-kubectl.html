CHEAT SHEET

kubectl

Referencia rapida

Cluster

Informacion del cluster

bash
kubectl cluster-info # URL del control plane y servicios kubectl get nodes # Lista de nodos del cluster kubectl version # Version del cliente y del servidor
Pods

Pods

bash
kubectl get pods # Listar pods del namespace actual kubectl get pods -o wide # Mas detalle: IP, nodo, etc. kubectl get pods -A # Todos los namespaces
bash
kubectl describe pod <nombre> # Detalle completo: eventos, estado, IP
bash
kubectl logs <nombre> # Ver logs del contenedor kubectl logs <nombre> -f # Seguir logs en tiempo real kubectl logs <nombre> --previous # Logs del contenedor anterior (crash)
bash
kubectl exec -it <nombre> -- bash # Shell interactivo dentro del pod
bash
kubectl port-forward pod/<nombre> 8080:80 # Acceder al pod en localhost:8080
bash
kubectl delete pod <nombre> # Eliminar un pod
Deployments

Deployments

bash
kubectl get deploy # Listar deployments kubectl describe deploy <nombre> # Detalle del deployment
bash
kubectl scale deploy <nombre> --replicas=3 # Escalar a 3 replicas
bash
kubectl rollout status deploy/<nombre> # Ver estado del rollout kubectl rollout undo deploy/<nombre> # Rollback a la version anterior
bash
kubectl set image deploy/<nombre> <container>=<imagen:tag> # Actualizar imagen
Services

Services

bash
kubectl get svc # Listar servicios kubectl describe svc <nombre> # Detalle: endpoints, puertos, tipo
bash
kubectl port-forward svc/<nombre> 8080:80 # Acceder al servicio en localhost
Configuracion

ConfigMaps y Secrets

ConfigMaps

bash
kubectl create configmap <nombre> --from-literal=KEY=value # Crear desde literal kubectl create configmap <nombre> --from-file=config.yaml # Crear desde archivo kubectl get cm # Listar ConfigMaps kubectl describe cm <nombre> # Ver contenido

Secrets

bash
kubectl create secret generic <nombre> --from-literal=PASS=s3cr3t # Crear secret kubectl get secrets # Listar secrets kubectl describe secret <nombre> # Ver metadatos (valores ocultos)
Namespaces

Namespaces

bash
kubectl get ns # Listar namespaces kubectl create ns <nombre> # Crear un namespace
bash
kubectl get pods -n <namespace> # Pods de un namespace especifico
bash
kubectl config set-context --current --namespace=<ns> # Cambiar namespace por defecto
Manifiestos

Apply y Delete

bash
kubectl apply -f archivo.yaml # Crear o actualizar recursos desde YAML kubectl apply -f ./directorio/ # Aplicar todos los YAML de un directorio
bash
kubectl delete -f archivo.yaml # Eliminar los recursos definidos en el YAML kubectl delete -f ./directorio/ # Eliminar todos los recursos del directorio
Debug

Debug rapido

Cuando algo no funciona, estos comandos son tu mejor aliado:

bash
kubectl get events --sort-by='.lastTimestamp' # Ver eventos recientes del cluster
bash
kubectl describe pod <nombre> # Revisar Events al final del output
bash
kubectl logs <nombre> --previous # Logs del contenedor que crasheo

Consejo: Si un pod esta en CrashLoopBackOff, revisa primero los logs y luego el describe.

Workflow

Flujo tipico

Flujo habitual de trabajo con kubectl:

1 Escribir el manifiesto YAML con los recursos necesarios (Deployment, Service, etc.)

2 Aplicar el manifiesto

bash
kubectl apply -f mi-app.yaml

3 Verificar que los pods estan corriendo

bash
kubectl get pods -w # -w para watch en tiempo real

4 Comprobar los logs

bash
kubectl logs deploy/mi-app

5 Acceder al servicio

bash
kubectl port-forward svc/mi-app 8080:80

6 Si algo falla, debug

bash
kubectl describe pod <nombre> kubectl logs <nombre> --previous kubectl get events
