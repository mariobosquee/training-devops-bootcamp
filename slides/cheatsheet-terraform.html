CHEAT SHEET

Terraform

Referencia rapida

1. Workflow basico

terminal
$ terraform init        # Descarga proveedores y modulos
$ terraform plan        # Vista previa de cambios (no toca nada)
$ terraform apply       # Aplica los cambios de verdad
$ terraform destroy     # Elimina toda la infraestructura creada

Regla de oro: siempre revisa el plan antes de hacer apply.

2. Estructura de archivos

Un proyecto Terraform tipico tiene esta estructura:

estructura
mi-proyecto/
├── main.tf              # Recursos principales
├── variables.tf         # Declaracion de variables
├── outputs.tf           # Valores de salida
├── terraform.tfvars     # Valores de las variables
├── .terraform/          # Proveedores descargados (no commitear)
└── terraform.tfstate    # Estado actual de la infra (no editar a mano)

.terraform/ y terraform.tfstate se generan automaticamente. Anade .terraform/ y *.tfstate* a tu .gitignore.

3. HCL basico

Los bloques fundamentales del lenguaje de configuracion de Terraform:

hcl
# --- PROVIDER: que plataforma usamos ---
provider "aws" {
  region = "eu-west-1"
}

# --- RESOURCE: lo que queremos crear ---
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
  tags = {
    Name = "mi-servidor"
  }
}

# --- VARIABLE: parametro de entrada ---
variable "entorno" {
  type        = string
  default     = "dev"
  description = "Nombre del entorno"
}

# --- OUTPUT: valor de salida ---
output "ip_publica" {
  value = aws_instance.web.public_ip
}

# --- DATA: leer datos existentes (sin crear nada) ---
data "aws_ami" "ubuntu" {
  most_recent = true
  owners      = ["099720109477"]
}

# --- LOCALS: variables calculadas internas ---
locals {
  nombre_completo = "${var.entorno}-web-server"
}

4. Variables

Tipos de variables disponibles y como usarlas:

hcl
# string
variable "nombre" {
  type    = string
  default = "mi-app"
}

# number
variable "puerto" {
  type    = number
  default = 8080
}

# bool
variable "produccion" {
  type    = bool
  default = false
}

# list
variable "zonas" {
  type    = list(string)
  default = ["eu-west-1a", "eu-west-1b"]
}

# map
variable "tags" {
  type = map(string)
  default = {
    equipo  = "devops"
    entorno = "dev"
  }
}

# --- Usar variables en un recurso ---
resource "local_file" "config" {
  content  = "App: ${var.nombre}, Puerto: ${var.puerto}"
  filename = "${var.nombre}.conf"
}

Prioridad de asignacion (de menor a mayor): default → terraform.tfvars → -var-file → -var → TF_VAR_*

5. State

Terraform guarda el estado de la infraestructura en terraform.tfstate. Comandos para gestionarlo:

terminal
# Listar todos los recursos en el estado
$ terraform state list
local_file.saludo
docker_container.nginx

# Ver detalle de un recurso concreto
$ terraform state show local_file.saludo
# resource "local_file" "saludo" {
#     content  = "Hola mundo"
#     filename = "saludo.txt"
#     id       = "abc123..."
# }

# Eliminar un recurso del estado (sin destruirlo)
$ terraform state rm local_file.saludo
Removed local_file.saludo

# Ver el estado completo en formato JSON
$ terraform show

Nunca edites el fichero terraform.tfstate a mano. Usa siempre los comandos terraform state.

7. Flags utiles

terminal
# Aplicar sin pedir confirmacion (cuidado en produccion)
$ terraform apply -auto-approve

# Pasar una variable desde la linea de comandos
$ terraform apply -var="entorno=prod"

# Usar un fichero de variables personalizado
$ terraform apply -var-file="produccion.tfvars"

# Aplicar solo a un recurso concreto
$ terraform apply -target=local_file.saludo

# Formatear el codigo HCL automaticamente
$ terraform fmt

# Validar la sintaxis sin ejecutar nada
$ terraform validate

8. Flujo tipico

Asi es el dia a dia trabajando con Terraform:

  1. Crear directorio del proyecto y escribir los ficheros .tf
  2. Ejecutar terraform init para descargar proveedores
  3. Ejecutar terraform fmt para formatear el codigo
  4. Ejecutar terraform validate para comprobar la sintaxis
  5. Ejecutar terraform plan y revisar los cambios propuestos
  6. Ejecutar terraform apply para crear la infraestructura
  7. Verificar que los recursos se han creado correctamente
  8. Commitear los ficheros .tf a Git (nunca el .tfstate)
  9. Cuando ya no necesites la infra: terraform destroy
