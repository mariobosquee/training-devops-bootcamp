Contexto

Por que WSL2

En el bootcamp usamos herramientas Linux: Docker, Kubernetes, Terraform, Helm. WSL2 te da un terminal Ubuntu real dentro de Windows.

  • Todos los comandos funcionan tal cual — sin adaptaciones para Windows
  • Docker Desktop se integra automaticamente con WSL2
  • VS Code se conecta directamente al entorno Linux
  • Aprendes Linux de paso — el SO dominante en servidores y cloud
Importante

Si ya usas Linux o macOS nativamente, puedes saltarte el paso de WSL2 e ir directamente a instalar las herramientas.

Paso 1 · 5 minutos

Instalar WSL2

1 Abrir PowerShell como Administrador:

PowerShell (Admin)
wsl --install -d Ubuntu

2 Reiniciar el ordenador cuando lo pida.

3 Al reiniciar, se abrira Ubuntu. Crear usuario y contrasena (no es la de Windows).

4 Verificar la version:

PowerShell
wsl --version
# Debe mostrar WSL version 2.x.x

5 Entrar en Ubuntu:

PowerShell
wsl
Nota

A partir de aqui, TODOS los comandos se ejecutan dentro de Ubuntu (WSL2), no en PowerShell.

Pasos 2-3 · 5 minutos

Ubuntu + Docker Desktop

1 Actualizar Ubuntu:

Ubuntu (WSL2)
sudo apt update && sudo apt upgrade -y

2 Instalar Docker Desktop desde docker.com (en Windows):

  • Descargar e instalar Docker Desktop para Windows
  • Activar "Use the WSL 2 based engine"
  • Settings → Resources → WSL Integration → Habilitar Ubuntu

3 Verificar dentro de Ubuntu:

Ubuntu (WSL2)
docker version
# Debe mostrar Client Y Server

docker run --rm hello-world
# Debe mostrar "Hello from Docker!"

Instalar Git

1 Instalar Git:

Ubuntu (WSL2)

Ubuntu (WSL2)

Paso 5 · 3 minutos

kubectl, kind, Helm

1 Instalar kubectl:

Ubuntu (WSL2)
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
chmod +x kubectl && sudo mv kubectl /usr/local/bin/

2 Instalar kind:

Ubuntu (WSL2)
curl -Lo kind https://kind.sigs.k8s.io/dl/v0.27.0/kind-linux-amd64
chmod +x kind && sudo mv kind /usr/local/bin/

3 Instalar Helm:

Ubuntu (WSL2)
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

4 Verificar los tres:

Ubuntu (WSL2)
kubectl version --client
kind version
helm version

Paso 6 · 2 minutos

Instalar Terraform

1 Anadir repositorio de HashiCorp:

Ubuntu (WSL2)
sudo apt install -y gnupg software-properties-common

wget -O- https://apt.releases.hashicorp.com/gpg | \
  gpg --dearmor | sudo tee /usr/share/keyrings/hashicorp-archive-keyring.gpg > /dev/null

echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] \
  https://apt.releases.hashicorp.com $(lsb_release -cs) main" | \
  sudo tee /etc/apt/sources.list.d/hashicorp.list

2 Instalar Terraform:

Ubuntu (WSL2)
sudo apt update && sudo apt install -y terraform

3 Verificar:

Ubuntu (WSL2)
terraform version
# Terraform v1.x.x

Paso 8 · 3 minutos

VS Code + WSL2

1 Abrir VS Code en Windows (descargalo de code.visualstudio.com si no lo tienes).

2 Instalar la extension "WSL" (de Microsoft):

  • Ir a Extensiones (Ctrl+Shift+X)
  • Buscar "WSL" y pulsar Instalar

3 Conectar a WSL:

  • Ctrl+Shift+P → escribir "WSL: Connect to WSL"
  • El terminal integrado de VS Code sera ahora Ubuntu
Truco

Desde el terminal Ubuntu puedes ejecutar code . para abrir VS Code directamente en la carpeta actual.

Paso 9 · 2 minutos

Clonar el repositorio

1 Crear la carpeta y clonar:

Ubuntu (WSL2)
mkdir -p ~/bootcamp && cd ~/bootcamp
git clone https://github.com/ORGANIZACION/training-devops-bootcamp.git
cd training-devops-bootcamp

2 Verificar el contenido:

Ubuntu (WSL2)
ls apps/
# campuseats  monitoring  myfitness  quizbattle

3 Abrir en VS Code:

Ubuntu (WSL2)
code .
Importante

Trabaja siempre dentro de ~/bootcamp/, NO en /mnt/c/. El rendimiento de Docker y Git es mucho mejor en el sistema de archivos nativo de Linux.

Ayuda

Problemas frecuentes

Si algo falla
wsl --install falla
Necesitas Windows 10 2004+ o Windows 11
WSL dice version 1
Ejecuta wsl --set-default-version 2
Docker no funciona en WSL2
Abre Docker Desktop → Settings → Resources → WSL Integration → Habilita Ubuntu
sudo pide contrasena
Es la contrasena de Ubuntu que creaste al instalar WSL, no la de Windows
curl / wget fallan
Comprueba tu conexion a internet. Si hay proxy, consulta al instructor
VS Code no conecta a WSL
Reinstala la extension WSL y reinicia VS Code
