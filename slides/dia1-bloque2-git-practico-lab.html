01 / 08
Preparacion

Antes de empezar

Objetivo: Confirmar que Git y GitHub funcionan correctamente antes de arrancar con Docker.

1 Verificar Git:

git --version
# Debeis ver: git version 2.43.0 (o superior)

2 Configurar nombre y email (si estan vacios):

git config --global user.name
git config --global user.email

# Si estan vacios, configurad:
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

Necesitais tambien: cuenta de GitHub y navegador abierto.

02 / 08
Ejercicio

Clonar repo y crear rama

1 Clonar el repositorio del bootcamp:

git clone https://github.com/cpalanca/bootcamp-devops-2026.git
cd bootcamp-devops-2026

2 Crear vuestra rama:

git checkout -b setup/tu-nombre

Reemplazad tu-nombre por vuestro nombre real (sin espacios, ej: setup/maria-garcia).

carlospalanca.es
03 / 08
Ejercicio

Crear archivo de presentacion

3 Crear la carpeta y el archivo:

mkdir -p alumnos
code alumnos/tu-nombre.md

Pegad esta plantilla y rellenadla:

# Tu Nombre

- **Grado / Master:** [tu titulacion]
- **Una cosa que espero aprender:** [lo que sea]
- **Sistema operativo:** [Windows / macOS / Linux]
- **Experiencia con terminal:** [ninguna / basica / media]

Guardad con Ctrl+S.

carlospalanca.es
04 / 08
Ejercicio

Commit, push y PR

4 Hacer commit y push:

git add alumnos/tu-nombre.md
git commit -m "feat: presentacion de tu-nombre"
git push -u origin setup/tu-nombre

5 Abrir Pull Request en GitHub:

  • Id al repositorio en el navegador
  • Vereis un banner «Compare & pull request» — pulsadlo
  • Titulo: setup: presentacion de tu-nombre
  • Pulsad Create pull request

6 Verificacion: si veis vuestra PR abierta en GitHub, todo funciona.

Troubleshooting

Si algo falla

Errores comunes
git: command not found
Instalad Git desde git-scm.com y reiniciad la terminal
Permission denied (publickey)
Usad HTTPS en vez de SSH: git clone https://...
failed to push
Comprobad que habeis hecho git checkout -b y no estais en main
No aparece el banner de PR
Id a la pestana Pull requests → New pull request y seleccionad vuestra rama
Pide usuario y contrasena
Usad un Personal Access Token en vez de contrasena (Settings → Developer settings → Tokens)
