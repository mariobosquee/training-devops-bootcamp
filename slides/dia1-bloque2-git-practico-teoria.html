Git Practico

Ramas, commits, merge, Pull Requests y flujo de trabajo con GitHub

1. Workflow del bootcamp

Durante estos 4 dias vamos a trabajar siempre con el mismo flujo:

Git workflow
1. Crear rama git checkout -b feature/nombre-descriptivo 2. Hacer cambios git add . && git commit -m "descripcion clara" 3. Subir rama git push -u origin feature/nombre-descriptivo 4. Abrir PR en GitHub (desde la web) 5. Companero revisa (comenta, aprueba) 6. Merge a main (boton en GitHub)

Convenciones que usaremos

ElementoConvencionEjemplo
Ramasfeature/descripcion-cortafeature/dockerfile-backend
CommitsImperativo, en espanol, sin punto finalAnade Dockerfile para la API
PRsTitulo corto + descripcion de que cambiaDockerizar backend con multi-stage

2. Cheat sheet rapido

ComandoQue hace
git statusVer estado (usadlo siempre antes de todo)
git add .Preparar todos los cambios
git commit -m "msg"Guardar cambios con mensaje
git pushSubir a GitHub
git pullDescargar cambios de GitHub
git checkout -b nombreCrear rama y moverte a ella
git checkout mainVolver a la rama principal
git merge nombreFusionar rama con la actual
git log --onelineHistorial resumido

3. Recordatorio: resolver conflictos

Si dos personas tocan la misma linea:

Conflicto de merge
<<<<<<< HEAD Tu version ======= Version del companero >>>>>>> feature
  1. Elige que texto conservar.
  2. Borra las marcas <<<<<<<, =======, >>>>>>>.
  3. git add + git commit.
No es un error. Es parte del flujo normal.
