Workflow

Workflow del bootcamp

Este flujo lo repetiremos en cada bloque, cada dia:

1. git checkout -b feature/nombre    # Crear rama
2. git add . && git commit -m "..."  # Guardar cambios
3. git push -u origin feature/nombre # Subir a GitHub
4. Abrir Pull Request en GitHub      # Pedir revision
5. Companero revisa y aprueba        # Code review
6. Merge a main                      # Fusionar

Ramasfeature/descripcion-cortafeature/dockerfile-backend
CommitsImperativo, en espanolAnade Dockerfile para la API
PRsTitulo corto + descripcionDockerizar backend

Cheat sheet

Cheat sheet

git status                # Estado actual (usadlo SIEMPRE)
git add .                 # Preparar cambios
git commit -m "mensaje"   # Guardar foto
git push                  # Subir a GitHub
git pull                  # Descargar cambios
git checkout -b nombre    # Crear rama y moverte
git checkout main         # Volver a main
git merge nombre          # Fusionar rama
git log --oneline         # Historial resumido

Conflictos: abrir archivo → elegir texto → borrar marcas → git add + git commit. No es un error, es normal.
