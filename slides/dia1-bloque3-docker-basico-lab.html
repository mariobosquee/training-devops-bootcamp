Parte 1

Crear el contenedor Nginx

1 Crear el contenedor:

docker run -d --name bootcamp-web -p 9999:80 nginx
  • -d — ejecuta en segundo plano (detached)
  • --name bootcamp-web — nombre concreto en vez de aleatorio
  • -p 9999:80 — conecta puerto 9999 de tu PC con el 80 del contenedor

Abrid el navegador: http://localhost:9999

Debeis ver la pagina de bienvenida de Nginx.

Parte 2

Crear volumen y arrancar MySQL

1 Crear un volumen:

docker volume create datasql
docker volume ls

2 Arrancar MySQL con el volumen:

docker run -d \
  --name mi-mysql \
  -v datasql:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=bootcamp2026 \
  -p 3306:3306 \
  mysql:8.1
  • -v datasql:/var/lib/mysql — monta el volumen donde MySQL guarda datos
  • -e MYSQL_ROOT_PASSWORD=bootcamp2026 — contrasena de root
  • -p 3306:3306 — expone el puerto de MySQL

Esperad unos segundos y comprobad: docker logs mi-mysql

Crear datos de prueba

Parte 2

Los datos persisten

5 Eliminad el contenedor completamente:

docker stop mi-mysql
docker rm mi-mysql
docker volume ls    # El volumen sigue ahi!

6 Cread un contenedor nuevo con el mismo volumen:

docker run -d \
  --name mi-mysql-2 \
  -v datasql:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=bootcamp2026 \
  -p 3306:3306 \
  mysql:8.1

7 Comprobad que los datos siguen ahi:

docker exec -it mi-mysql-2 mysql -uroot -pbootcamp2026 \
  -e "SELECT * FROM bootcamp.alumnos;"
Clave

Los datos han sobrevivido a la destruccion del contenedor. Esto es fundamental para bases de datos en produccion.

Parte 2

Backup y restore

Crear un backup del volumen:

docker run --rm \
  -v datasql:/source \
  -v $(pwd):/backup \
  busybox tar czf /backup/mysql-backup.tar.gz -C /source .

ls -lh mysql-backup.tar.gz

Restaurar en un volumen nuevo:

docker volume create datasql_restaurado

docker run --rm \
  -v datasql_restaurado:/target \
  -v $(pwd):/backup \
  busybox tar xzf /backup/mysql-backup.tar.gz -C /target

Verificar la restauracion:

docker run -d --name mi-mysql-3 \
  -v datasql_restaurado:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=bootcamp2026 -p 3307:3306 mysql:8.1

# Esperad unos segundos y comprobad:
docker exec -it mi-mysql-3 mysql -uroot -pbootcamp2026 \
  -e "SELECT * FROM bootcamp.alumnos;"

Parte 3

Crear red y contenedores

1 Crear red y volumen:

docker network create red_bootcamp
docker volume create redis_data
docker network ls

2 Arrancar Redis en la red:

docker run -d \
  --name redis \
  --network red_bootcamp \
  -v redis_data:/data \
  redis

3 Arrancar dos servidores web Apache:

docker run -d --name web1 --network red_bootcamp -p 8081:80 httpd:2.4
docker run -d --name web2 --network red_bootcamp -p 8082:80 httpd:2.4
docker ps

Visitad http://localhost:8081 y http://localhost:8082

Parte 3

Resolucion DNS

4 Lanzad un contenedor para probar la red:

docker run -it --rm --network red_bootcamp busybox

Dentro del contenedor, haced ping por nombre:

ping -c 3 redis
ping -c 3 web1
ping -c 3 web2
exit
Clave

Docker resuelve los nombres automaticamente dentro de la misma red. No necesitais hardcodear IPs.

Parte 3

Comunicacion entre servicios

5 Conectaos a Redis por nombre DNS:

docker run -it --rm --network red_bootcamp redis redis-cli -h redis

Dentro del cliente Redis:

SET bootcamp "DevOps 2026"
GET bootcamp
SET equipo "Nuestro equipo"
GET equipo
KEYS *
exit

El cliente Redis no necesita saber la IP del servidor. Solo usa el nombre del contenedor.

6 Limpieza:

docker stop redis web1 web2
docker rm redis web1 web2
docker network rm red_bootcamp
docker volume rm redis_data

Parte 4

Crear infraestructura

1 Crear red y volumenes:

docker network create red_cloud
docker volume create nextcloud_data
docker volume create mariadb_data

2 Desplegar MariaDB:

docker run -d \
  --name mariadb \
  --network red_cloud \
  -v mariadb_data:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=secret \
  -e MYSQL_DATABASE=nextdb \
  -e MYSQL_USER=next_user \
  -e MYSQL_PASSWORD=next_secret \
  mariadb
  • MYSQL_DATABASE=nextdb — crea la BD automaticamente al arrancar
  • MYSQL_USER / MYSQL_PASSWORD — crea un usuario con acceso

Comprobad: docker logs mariadb

carlospalanca.es
19 / 24
Parte 4

Desplegar Nextcloud

3 Desplegar Nextcloud:

docker run -d \
  --name nextcloud \
  --network red_cloud \
  -v nextcloud_data:/var/www/html \
  -e MYSQL_HOST=mariadb \
  -e MYSQL_DATABASE=nextdb \
  -e MYSQL_USER=next_user \
  -e MYSQL_PASSWORD=next_secret \
  -p 8080:80 \
  nextcloud
Fijaos

MYSQL_HOST=mariadb — Nextcloud se conecta a MariaDB usando el nombre del contenedor como hostname. Funciona porque estan en la misma red Docker.

4 Verificar: esperad 15-20 segundos y abrid http://localhost:8080

Cread una cuenta de administrador con cualquier usuario y contrasena.

Parte 4

Explorar y personalizar

5 Investigad por vuestra cuenta:

  • Subid un archivo a Nextcloud desde el navegador
  • Comprobad los logs: docker logs nextcloud y docker logs mariadb
  • Inspeccionad la red: docker network inspect red_cloud
  • Comprobad los volumenes: docker volume inspect nextcloud_data
Reflexion

Dos servicios en contenedores separados, comunicandose por DNS, con datos persistentes en volumenes. Todo levantado con comandos simples, sin instalar nada en vuestro SO.

Dejad Nextcloud corriendo para el showcase.

Si algo falla
Pagina en blanco
Esperad 30s y refrescad. Comprobad docker logs nextcloud
Error de base de datos
MariaDB no ha arrancado. Esperad y reiniciad: docker restart nextcloud
Puerto 8080 en uso
Cambiad a -p 8888:80 y visitad :8888
