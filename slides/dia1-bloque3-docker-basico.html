Dockerfile: FROM y WORKDIR

# Dockerfile: la receta de tu imagen

### FROM -- La base

FROM node:18-alpine

"Empieza con una imagen que ya tiene Node.js 18 instalado."

Alpine es una version minima de Linux (5 MB). Perfecta para contenedores.

### WORKDIR -- Donde trabajar

WORKDIR /app

"Crea la carpeta /app y trabaja ahi dentro."

Dockerfile: COPY y R

Dockerfile: COPY y RUN

# COPY y RUN

### COPY -- Meter archivos en la imagen

COPY package*.json ./

COPY src/ ./src/

Copia archivos desde tu ordenador al interior de la imagen.

### RUN -- Ejecutar durante la construccion

RUN npm install

Instala las dependencias. Se ejecuta UNA vez, al construir la imagen.

Truco: copiar primero package.json y luego el codigo permite que Docker cache las dependencias.

Dockerfile: EXPOSE y

Dockerfile: EXPOSE y CMD

# EXPOSE y CMD

### EXPOSE -- Documentar el puerto

EXPOSE 3001

Informa de que la app usa el puerto 3001. Es documentacion, no abre el puerto.

### CMD -- El comando de arranque

CMD ["node", "src/index.js"]

Define que se ejecuta al iniciar el contenedor. Solo uno por Dockerfile.

### Dockerfile completo de nuestra API:

FROM node:18-alpine

WORKDIR /app


COPY package*.json ./


RUN npm install


COPY src/ ./src/


EXPOSE 3001


CMD ["node", "src/index.js"]

docker build y

docker build y docker run

# Construir y ejecutar

### Construir la imagen

docker build -t bootcamp-api .
  • `-t bootcamp-api` -- nombre de la imagen
  • `.` -- busca el Dockerfile en el directorio actual

### Ejecutar un contenedor

docker run -p 3001:3001 bootcamp-api
  • `-p 3001:3001` -- conecta el puerto de tu ordenador con el del contenedor
  • Formato: `-p PUERTO_TU_PC:PUERTO_CONTENEDOR`

Anade `-d` para ejecutar en segundo plano.

# Gestionar contenedores

### Ver contenedores activos

docker logs y d

docker logs y docker exec

# Depurar contenedores

### Ver logs

docker logs NOMBRE_O_ID

docker logs -f NOMBRE_O_ID    # seguir en tiempo real

Muestra lo que la aplicacion ha escrito en consola. Fundamental para encontrar errores.

### Entrar en un contenedor

docker exec -it NOMBRE_O_ID sh

Abre una terminal dentro del contenedor. Puedes explorar archivos, comprobar que hay instalado, etc.

Docker Compose: que

Docker Compose: que es

# Docker Compose

Una aplicacion real tiene varios servicios: API, frontend, base de datos...

Ejecutar cada uno con `docker run` y sus parametros es tedioso y propenso a errores.

Docker Compose permite definir todos los servicios en un archivo `docker-compose.yml` y levantarlos con un solo comando. 
Sin Compose:                    Con Compose:

docker build -t api .           docker compose up -d


docker build -t front .


docker run -p 3001:3001 api


docker run -p 3000:80 front
