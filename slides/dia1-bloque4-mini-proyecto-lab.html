Paso 1

Crear Dockerfile del backend

1 Explorad la estructura de la app:

cd backend
ls -la

2 Crear backend/Dockerfile:

FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY src/ ./src/

EXPOSE 3000

CMD ["node", "src/index.js"]

3 Crear backend/.dockerignore:

node_modules
npm-debug.log
.git

# Limpiar:
docker stop front-test && docker rm front-test
Nota

El frontend todavia no se comunica con el backend. Eso lo resolveremos con Docker Compose en el siguiente paso.

Paso 3

Crear docker-compose.yml

1 Crear docker-compose.yml en la raiz del proyecto:

services:
  api:
    build: ./backend
    ports:
      - "3000:3000"
    networks:
      - app-network
    volumes:
      - api-data:/app/data

  frontend:
    build: ./frontend
    ports:
      - "8080:80"
    networks:
      - app-network
    depends_on:
      - api

networks:
  app-network:

volumes:
  api-data:

Paso 3

Levantar y verificar

2 Levantar todo:

3 Verificar:

Clave

Con un solo comando (docker compose up) habeis levantado toda la aplicacion: backend, frontend, red y volumenes.

Si algo falla
Puerto en uso
Parad contenedores anteriores: docker stop $(docker ps -q)
Build falla
Revisad los Dockerfiles. Comprobad docker compose logs api

Paso 4

Comparar y optimizar

1 Ver el tamano actual:

docker images | grep bootcamp

2 Crear Dockerfile multi-stage para el backend (backend/Dockerfile):

# Etapa 1: Instalar dependencias
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install --production

# Etapa 2: Imagen final ligera
FROM node:18-alpine
WORKDIR /app
COPY --from=builder /app/node_modules ./node_modules
COPY src/ ./src/
EXPOSE 3000
CMD ["node", "src/index.js"]

3 Reconstruir y comparar:

docker compose build api
docker images | grep bootcamp
# Comparad el tamano antes y despues

Paso 5 · 5 minutos

Push a GitHub

1 Commit y push de vuestro trabajo:

git add backend/Dockerfile backend/.dockerignore
git add frontend/Dockerfile
git add docker-compose.yml
git commit -m "feat: dockerizar app con compose"
git push
Importante

No subais node_modules/ ni imagenes Docker al repositorio. El .dockerignore se encarga de excluirlos del build, y el .gitignore del repo.
