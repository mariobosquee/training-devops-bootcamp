GitHub Actions

Workflows, jobs, steps, actions del marketplace y GHCR

1. Que es GitHub Actions

GitHub Actions es la herramienta de CI/CD integrada directamente en GitHub. No hay que instalar nada, no hay que configurar servidores, no hay que crear cuentas en servicios externos.

Caracteristicas principales:

  • Viene incluido con cualquier repositorio de GitHub.
  • Gratis para repositorios publicos (minutos ilimitados).
  • Se configura con un archivo YAML dentro del propio repositorio.
  • Tiene un marketplace con miles de acciones reutilizables.

2. Conceptos clave

GitHub Actions tiene cinco conceptos fundamentales. Piensa en ellos como una jerarquia:

Jerarquia
Workflow (flujo de trabajo) | +-- Job (trabajo) | +-- Step (paso) | +-- Action (accion reutilizable) o comando
ConceptoQue esEjemplo
WorkflowUn proceso automatizado completo. Se define en un archivo YAML."Pipeline de CI"
JobUn grupo de pasos que se ejecutan en la misma maquina virtual."build", "test", "deploy"
StepUna tarea individual dentro de un job."Instalar dependencias", "Ejecutar tests"
ActionUn paso reutilizable creado por la comunidad o por GitHub.actions/checkout@v4
TriggerEl evento que dispara el workflow.on: push

3. Donde vive el workflow

Los workflows se guardan en una carpeta especifica dentro del repositorio:

Estructura
mi-repositorio/ .github/ workflows/ ci.yml <-- Aqui va nuestro pipeline backend/ frontend/ docker-compose.yml

GitHub detecta automaticamente cualquier archivo .yml dentro de .github/workflows/ y lo ejecuta segun sus triggers.

4. Anatomia de un workflow.yml

Este es un ejemplo completo con anotaciones:

.github/workflows/ci.yml
# Nombre del workflow (aparece en la pestana Actions) name: CI Pipeline # Trigger: cuando se ejecuta este workflow on: push: branches: [main] # Solo cuando se hace push a main # Jobs: los trabajos que se ejecutan jobs: build-and-test: # Nombre del job (tu eliges) runs-on: ubuntu-latest # Maquina virtual donde se ejecuta # Steps: los pasos del job steps: # Paso 1: Descargar el codigo del repositorio - name: Checkout codigo uses: actions/checkout@v4 # Paso 2: Instalar Node.js - name: Setup Node.js uses: actions/setup-node@v4 with: node-version: '18' # Paso 3: Instalar dependencias - name: Instalar dependencias run: npm install working-directory: ./backend # Paso 4: Ejecutar tests - name: Ejecutar tests run: npm test working-directory: ./backend

Desglose linea por linea

  • name: — Nombre descriptivo que aparece en la interfaz de GitHub.
  • on: — Define cuando se dispara. push a main es lo mas comun para empezar.
  • jobs: — Contiene uno o mas jobs. Cada job se ejecuta en su propia maquina virtual.
  • runs-on: — Sistema operativo. Usamos ubuntu-latest (Linux).
  • steps: — Lista ordenada de pasos. Se ejecutan uno tras otro.
  • uses: — Usa una action del marketplace (por ejemplo, actions/checkout@v4).
  • run: — Ejecuta un comando de terminal (por ejemplo, npm install).
  • working-directory: — Cambia la carpeta antes de ejecutar el comando.

5. Triggers mas comunes

TriggerCuando se ejecuta
on: pushEn cada push a cualquier rama
on: push: branches: [main]Solo en push a main
on: pull_requestCuando se abre o actualiza un pull request
on: workflow_dispatchManualmente desde la interfaz de GitHub

Para este bootcamp usaremos on: push: branches: [main].

6. Secrets: variables seguras

Algunos pasos necesitan credenciales (por ejemplo, para subir imagenes Docker). GitHub Actions proporciona:

  • GITHUB_TOKEN — Un token automatico que permite interactuar con el propio repositorio. No hay que configurarlo, ya viene incluido.
  • Secrets personalizados — Se configuran en Settings > Secrets del repositorio.

Para usar un secret en el workflow:

yaml
env: MI_SECRET: ${{ secrets.MI_SECRET }}

El GITHUB_TOKEN se accede con ${{ secrets.GITHUB_TOKEN }}.

7. Resumen

  • GitHub Actions esta integrado en GitHub. No hay que instalar nada.
  • Un workflow se define en .github/workflows/nombre.yml.
  • Un workflow tiene jobs, cada job tiene steps.
  • Los triggers definen cuando se ejecuta (por ejemplo, en cada push).
  • Las actions son pasos reutilizables del marketplace.
  • El GITHUB_TOKEN permite interactuar con GitHub sin configuracion adicional.
