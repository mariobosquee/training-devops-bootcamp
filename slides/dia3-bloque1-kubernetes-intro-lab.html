Parte 1

Crear y verificar el cluster

1 Crear el cluster:

kind create cluster --name bootcamp
# Tarda 1-2 minutos. Esperad a ver "Have a nice day!"

2 Verificar que el cluster esta listo:

kubectl cluster-info
# Debeis ver: Kubernetes control plane is running at https://...

3 Ver los nodos del cluster:

kubectl get nodes
# Debeis ver un nodo con STATUS: Ready
Si algo falla
"Cannot connect to Docker daemon"
Arrancad Docker Desktop y esperad a que este listo
"kind: command not found"
Instalad kind: choco install kind (Windows) o brew install kind (Mac)
El cluster ya existe
Eliminadlo: kind delete cluster --name bootcamp y volved a crearlo

Parte 2

Lanzar y explorar el pod

1 Crear un pod con nginx:

kubectl run nginx --image=nginx

2 Ver que el pod esta corriendo:

kubectl get pods
# Esperad hasta que STATUS sea Running

3 Inspeccionar el pod:

kubectl describe pod nginx
# Vereis eventos, IP, contenedor, imagen...

4 Ver los logs:

kubectl logs nginx

5 Acceder desde el navegador:

kubectl port-forward pod/nginx 8080:80
# Abrid http://localhost:8080 — debeis ver la pagina de Nginx
# Parad con Ctrl+C

6 Eliminar el pod:

kubectl delete pod nginx
kubectl get pods    # Debeis ver: No resources found
Si algo falla
Pod en ImagePullBackOff
Comprobad conexion a internet. Esperad y haced kubectl describe pod nginx
Puerto 8080 en uso
Usad otro puerto: kubectl port-forward pod/nginx 9090:80
port-forward se cierra
Comprobad que el pod sigue Running con kubectl get pods
