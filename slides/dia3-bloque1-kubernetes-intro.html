El problema

El problema

Gestionar muchos contenedores a mano es imposible.
  • 10 microservicios x 3 replicas = 30 contenedores.
  • Se cae uno a las 3 AM... quien lo reinicia?
  • Llega mas trafico... quien lanza mas replicas?
  • Nueva version... quien para los viejos y arranca los nuevos?

`docker run` no escala.

La solucion: Kuberne

La solucion: Kubernetes

Kubernetes (K8s) es un orquestador de contenedores.

  • Despliega contenedores automaticamente.
  • Escala segun la carga.
  • Se recupera de fallos solo (self-healing).
  • Actualiza sin downtime (rolling updates).

Analogia

Analogia: el director de hotel

Tu (kubectl)  -->  Director (K8s)  -->  Habitaciones (Pods)


    

  • "Quiero 3 habitaciones dobles"  -->  El director las asigna.
    • 

  • Una habitacion se inunda         -->  El director reubica al huesped.
    • 

  • Llegan 50 turistas               -->  El director abre mas habitaciones.
    • 

  • Renovacion de una planta          -->  El director mueve huespedes sin que se enteren.

Tu dices QUE quieres. Kubernetes decide COMO hacerlo.

Pod

Pod

El Pod es la unidad minima de Kubernetes.

+--- Pod ---+

Contenedor
(nginx)
+------------+
  • Normalmente 1 contenedor por pod.
  • Tiene su propia IP dentro del cluster.
  • Es efimero: si se muere, se crea uno nuevo.

Deployment

Deployment

El Deployment gestiona un grupo de pods.

Deployment: bootcamp-api (replicas: 3)

|


+-- Pod 1 [Running]


+-- Pod 2 [Running]


+-- Pod 3 [Running]

  • Tu dices cuantas replicas quieres.
  • Si un pod se cae, el Deployment crea otro.
  • Gestiona rolling updates y rollbacks.

Service

Service

El Service da una direccion estable a los pods.

Cliente  -->  Service (IP fija)  -->  Pod 1

-->  Pod 2


-->  Pod 3

  • Los pods van y vienen (IPs cambian).
  • El Service siempre apunta a los pods correctos.
  • Hace balanceo de carga automatico.

ConfigMap

ConfigMap

El ConfigMap almacena configuracion fuera del contenedor.

apiVersion: v1

kind: ConfigMap


metadata:


name: bootcamp-config


data:


NODE_ENV: production


PORT: "3333"

  • Variables de entorno, archivos de configuracion.
  • Separas la config de la imagen Docker.
  • Puedes cambiar la config sin reconstruir la imagen.

kind

kind: Kubernetes en tu portatil

kind = Kubernetes IN Docker.
  • Crea un cluster de K8s dentro de contenedores Docker.
  • Perfecto para desarrollo y aprendizaje.
  • Un solo comando para tener un cluster listo.
kind create cluster --name bootcamp
