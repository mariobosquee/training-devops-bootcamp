Parte 1

Instalar y verificar

4 Instalar nginx:

helm install mi-nginx bitnami/nginx

5 Verificar que se ha desplegado:

kubectl get pods
# Debeis ver: mi-nginx-xxxx  STATUS: Running (esperad 1-2 min)

kubectl get svc
# Debeis ver: mi-nginx  con un puerto asignado

6 Ver los releases de Helm:

helm list
# Debeis ver: mi-nginx  con STATUS: deployed
Fijaos

Con un solo comando, Helm ha creado un Deployment, un Service, un ConfigMap y mas. Sin escribir ningun YAML.

Si algo falla
"repo already exists"
No pasa nada, continuad con helm repo update
Pod en Pending
Esperad 1-2 minutos. El nodo de kind es lento descargando imagenes
"cannot re-use a name"
Ya existe un release con ese nombre: helm uninstall mi-nginx e intentad de nuevo

Parte 2

Crear y aplicar values

1 Crear values-custom.yaml:

replicaCount: 3

service:
  type: ClusterIP

2 Aplicar con helm upgrade:

helm upgrade mi-nginx bitnami/nginx -f values-custom.yaml

3 Verificar que ahora hay 3 replicas:

kubectl get pods
# Debeis ver 3 pods de mi-nginx (esperad a que esten Running)

4 Ver el historial de Helm:

helm history mi-nginx
# Debeis ver 2 revisiones: la original y la actualizada
Si algo falla
Solo 1 pod Running
Esperad 1-2 min. Comprobad: kubectl get pods -w
Error de YAML en values
Revisad la indentacion. replicaCount va sin espacios delante

Parte 3

Rollback y limpiar

1 Hacer rollback a la revision 1 (1 replica):

helm rollback mi-nginx 1

2 Verificar que volvemos a 1 pod:

kubectl get pods
# Debeis ver solo 1 pod de mi-nginx

3 Ver el historial completo (3 revisiones):

helm history mi-nginx
# Revision 1: install
# Revision 2: upgrade (3 replicas)
# Revision 3: rollback to 1

4 Desinstalar completamente:

helm uninstall mi-nginx

5 Verificar que se ha limpiado todo:

helm list
# No debeis ver mi-nginx

kubectl get pods
# Los pods de mi-nginx han desaparecido
Clave

Helm gestiona todo el ciclo de vida: install, upgrade, rollback, uninstall. Un solo comando para cada operacion.

Si algo falla
Rollback no reduce pods
Esperad 30s. Los pods tardan en terminar: kubectl get pods -w
Uninstall deja recursos
Comprobad PVCs: kubectl get pvc y eliminadlos manualmente si quedan
