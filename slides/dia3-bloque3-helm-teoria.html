Helm

El gestor de paquetes de Kubernetes

Que es Helm

Imagina que quieres instalar un programa en tu ordenador. En Ubuntu usas apt install, en Mac usas brew install. No te preocupas de donde van los ficheros, las dependencias ni la configuracion inicial: el gestor de paquetes se encarga de todo.

Helm es exactamente eso, pero para Kubernetes. En lugar de instalar programas en un ordenador, instalas aplicaciones en un cluster de Kubernetes con un solo comando.

Sin Helm, desplegar una aplicacion en Kubernetes requiere crear y mantener multiples ficheros YAML: el Deployment, el Service, el ConfigMap, el Ingress... Para una aplicacion sencilla pueden ser 5-10 ficheros. Para una aplicacion compleja, facilmente 20 o mas. Modificar un valor (por ejemplo, el numero de replicas) implica buscar y editar el fichero correcto a mano.

Con Helm, toda esa complejidad se empaqueta en un unico "paquete" que se instala, actualiza y desinstala con un solo comando.

Conceptos clave

Helm maneja cuatro conceptos fundamentales:

ConceptoQue esAnalogia
ChartUn paquete que contiene todos los YAML de una aplicacionComo un .deb o un .pkg
ReleaseUna instancia instalada de un Chart en tu clusterComo un programa ya instalado
RepositoryUn almacen donde se publican ChartsComo un repositorio de apt o brew
ValuesLos parametros de configuracion de un ChartComo las opciones de instalacion

Un mismo Chart (por ejemplo, bitnami/nginx) se puede instalar varias veces con distintos nombres y configuraciones. Cada instalacion es un Release diferente.

Comandos basicos

Estos son los comandos que usaras en el laboratorio:

# Anadir un repositorio de Charts
helm repo add bitnami https://charts.bitnami.com/bitnami

# Buscar Charts disponibles
helm search repo bitnami/nginx

# Instalar un Chart (crear un Release)
helm install mi-nginx bitnami/nginx

# Ver los Releases instalados
helm list

# Actualizar un Release con nuevos valores
helm upgrade mi-nginx bitnami/nginx -f values-custom.yaml

# Ver el historial de versiones de un Release
helm history mi-nginx

# Volver a una version anterior
helm rollback mi-nginx 1

# Desinstalar un Release
helm uninstall mi-nginx

Donde encontrar Charts

ArtifactHub (https://artifacthub.io) es el buscador publico de Charts de Helm. Ahi puedes encontrar Charts para casi cualquier aplicacion: nginx, PostgreSQL, Redis, Grafana, ArgoCD y miles mas. Cada Chart incluye documentacion sobre los valores que puedes configurar.

Resumen

  • Helm empaqueta aplicaciones de Kubernetes en Charts reutilizables.
  • Instalas, actualizas y desinstalas aplicaciones con un solo comando.
  • Personalizas la configuracion mediante ficheros de Values sin tocar los YAML originales.
  • No necesitas crear Charts desde cero: la comunidad ya tiene Charts para casi todo.
