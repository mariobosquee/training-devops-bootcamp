Que es Helm

Que es Helm

Helm = apt/brew para Kubernetes
  • En Ubuntu: `apt install nginx` instala nginx en tu ordenador
  • En Kubernetes: `helm install mi-nginx bitnami/nginx` instala nginx en tu cluster

Un solo comando empaqueta, instala y configura toda la aplicacion.

Conceptos clave (1/2

Conceptos clave (1/2)

Chart y Release
  • Chart: paquete que contiene todos los YAML de una aplicacion (como un .deb)
  • Release: una instancia instalada de un Chart en tu cluster (como un programa instalado)

Un mismo Chart se puede instalar varias veces con distintos nombres.

Conceptos clave (2/2

Conceptos clave (2/2)

Repository y Values
  • Repository: almacen donde se publican Charts (como un repositorio de apt)
  • Values: parametros de configuracion que personalizan un Chart (replicas, puertos, recursos...)

No tocas los YAML originales: solo cambias los Values.

Comandos principales

Comandos principales

helm repo add       Anadir un repositorio

helm search repo    Buscar Charts


helm install        Instalar un Chart


helm list           Ver Releases instalados


helm upgrade        Actualizar un Release


helm rollback       Volver a una version anterior


helm uninstall      Eliminar un Release

ArtifactHub

ArtifactHub

https://artifacthub.io

El buscador publico de Charts de Helm.

  • Miles de Charts listos para usar: nginx, PostgreSQL, Redis, Grafana, ArgoCD...
  • Cada Chart incluye documentacion de todos los Values configurables.
  • Mantenidos por la comunidad y por empresas como Bitnami.

Resumen

Resumen

Lo que hemos aprendido
  • Helm empaqueta aplicaciones de Kubernetes en Charts reutilizables.
  • Instalamos, actualizamos y desinstalamos con un solo comando.
  • Personalizamos la configuracion con ficheros de Values.
  • No creamos Charts desde cero: usamos los que ya existen en ArtifactHub.
