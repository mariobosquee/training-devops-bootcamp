Parte 1

Instalar ArgoCD en el cluster

1 Crear el namespace y aplicar la instalacion:

kubectl create namespace argocd

kubectl apply -n argocd \
  -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

2 Esperar a que los pods esten listos:

kubectl wait --for=condition=Ready pods --all -n argocd --timeout=120s

kubectl get pods -n argocd
# Debeis ver 5-7 pods en estado Running

3 Obtener la contrasena del admin:

kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath="{.data.password}" | base64 -d
# Copiad la contrasena que aparece

kubectl port-forward svc/argocd-server -n argocd 8443:443

Parte 3

Cambiar replicas en Git

1 Editad k8s/backend-deployment.yaml y cambiad las replicas:

# Cambiad replicas: 1 a replicas: 2
spec:
  replicas: 2

2 Commit y push:

git add k8s/backend-deployment.yaml
git commit -m "scale backend to 2 replicas"
git push

3 En ArgoCD UI: click en REFRESH — la app aparecera como OutOfSync (amarillo).

4 Click en SYNC para aplicar el cambio.

5 Verificar:

kubectl get pods
# Debeis ver 2 pods del backend
GitOps en accion

No habeis tocado kubectl para escalar. El cambio fue en Git y ArgoCD lo aplico al cluster. Esto es GitOps.
