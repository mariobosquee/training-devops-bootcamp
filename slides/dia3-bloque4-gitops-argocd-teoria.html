GitOps con ArgoCD

Git como fuente de verdad para Kubernetes

Que es GitOps

Hasta ahora, para desplegar cambios en Kubernetes habeis ejecutado comandos manualmente: kubectl apply, helm install, etc. Esto funciona, pero tiene problemas:

  • No queda registro claro de quien cambio que y cuando.
  • Si alguien ejecuta un comando incorrecto, es dificil volver atras.
  • Cada persona del equipo puede tener una version diferente de los ficheros YAML.

GitOps resuelve esto con una idea sencilla: Git es la unica fuente de verdad. Todo lo que debe estar en el cluster se define en un repositorio Git. Si quieres cambiar algo en el cluster, no ejecutas comandos: haces un commit en Git. Una herramienta automatica detecta el cambio y actualiza el cluster por ti.

Analogia: Google Docs

Pensad en Google Docs. Cuando alguien edita un documento, todos los demas ven el cambio al instante. No hace falta enviar el fichero por correo ni avisar a nadie. El documento compartido es la fuente de verdad y todos trabajan sobre el.

GitOps funciona igual: el repositorio Git es el "documento compartido". Cuando alguien hace un cambio (commit + push), el cluster se actualiza automaticamente para reflejar ese cambio.

Push vs Pull

Hay dos modelos para aplicar los cambios de Git al cluster:

ModeloComo funciona
PushUn sistema externo (como una pipeline de CI/CD) ejecuta kubectl apply despues de cada commit. El sistema "empuja" los cambios al cluster.
PullUna herramienta dentro del cluster vigila el repositorio Git continuamente. Cuando detecta un cambio, ella misma "tira" de los nuevos ficheros y los aplica.

El modelo Pull es mas seguro porque:

  • Nadie externo necesita credenciales del cluster.
  • Si alguien modifica algo en el cluster manualmente, la herramienta lo detecta y lo corrige.

Que es ArgoCD

ArgoCD es una herramienta GitOps que funciona con el modelo Pull. Se instala dentro del cluster de Kubernetes y hace lo siguiente:

  • Vigila un repositorio Git que contiene los manifiestos YAML de tu aplicacion.
  • Compara el estado del repositorio con el estado real del cluster.
  • Sincroniza el cluster para que coincida con Git si hay diferencias.

ArgoCD incluye una interfaz web donde puedes ver:

  • Todas las aplicaciones desplegadas.
  • Si estan sincronizadas con Git o no.
  • El estado de cada recurso (pods, servicios, deployments, etc.).
  • Un mapa visual de los recursos y sus relaciones.

Resumen

  • GitOps = Git es la unica fuente de verdad. Cambias Git, cambia el cluster.
  • Modelo Pull = ArgoCD vigila Git desde dentro del cluster y aplica los cambios.
  • ArgoCD = herramienta GitOps con interfaz web que sincroniza Kubernetes con Git.
  • No ejecutas kubectl apply manualmente: haces commit y ArgoCD se encarga.
