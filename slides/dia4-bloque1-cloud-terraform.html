El problema

El problema: infraestructura manual

Crear infraestructura haciendo clic en la consola web:

  • Lento (50 recursos = horas de clics)
  • Errores (un clic mal = configuracion rota)
  • No reproducible (repetir en otro entorno = repetir todos los clics)
  • Sin historial (quien cambio que?)

Mismo problema que el codigo antes de Git.

carlospalanca.es
06 / 11
La solucion

La solucion: Infrastructure as Code

Escribir la infraestructura en ficheros de texto.

  • Versionado en Git (quien cambio que y cuando)
  • Reproducible (mismo codigo = misma infraestructura)
  • Revisable (pull request antes de aplicar cambios)
  • Automatizable (CI/CD puede ejecutar Terraform)

Que es Terraform

Que es Terraform

Herramienta de HashiCorp para crear infraestructura desde codigo.

  • Multi-cloud: AWS, Azure, GCP, Docker, Kubernetes...
  • Declarativo: describes QUE quieres, no COMO hacerlo
  • Estado: recuerda lo que ha creado (terraform.tfstate)
  • Plan: te ensena los cambios antes de aplicarlos

HCL: ejemplo basico

}

4 bloques clave: terraform, provider, resource, variable.

El flujo de Terrafor

El flujo de Terraform

terraform init      Descarga proveedores

|


v


terraform plan      Vista previa (no toca nada)


|


v


terraform apply     Ejecuta los cambios


|


v


terraform destroy   Elimina todo

Regla de oro: NUNCA apply sin mirar el plan.
