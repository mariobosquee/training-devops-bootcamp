Estructura web con Terraform

Crear archivos locales con Infrastructure as Code

carlospalanca.es
04 / 18
Parte 1

Crear el proyecto Terraform

1 Crear el directorio del laboratorio:

mkdir -p ~/terraform-lab && cd ~/terraform-lab

2 Crear main.tf con el provider local y recursos para una estructura web:

terraform {
  required_providers {
    local = {
      source  = "hashicorp/local"
      version = "~> 2.0"
    }
  }
}

resource "local_file" "index" {
  content  = "<h1>Bootcamp DevOps 2026</h1>\n<p>Generado con Terraform</p>"
  filename = "${path.module}/website/index.html"
}

resource "local_file" "styles" {
  content  = "body { font-family: sans-serif; text-align: center; }"
  filename = "${path.module}/website/styles.css"
}

resource "local_file" "readme" {
  content  = "# Mi Website\nGenerado automaticamente con Terraform."
  filename = "${path.module}/website/README.md"
}

Parte 1

Init, Plan, Apply

3 Inicializar Terraform:

terraform init
# Descarga el provider local

4 Ver el plan de ejecucion:

terraform plan
# Debeis ver: Plan: 3 to add, 0 to change, 0 to destroy.

5 Aplicar:

terraform apply
# Escribid "yes" cuando pregunte

6 Verificar y destruir:

find website/ -type f
# Debeis ver los 3 archivos creados

terraform destroy
# Escribid "yes" — los archivos desaparecen
Clave

El ciclo de Terraform: init → plan → apply → destroy. Siempre revisad el plan antes de aplicar.

Parte 2

Definir variables

1 Crear variables.tf:

variable "nombre_entorno" {
  description = "Nombre del entorno"
  type        = string
  default     = "bootcamp"
}

variable "proyectos" {
  description = "Lista de proyectos"
  type        = list(string)
  default     = ["frontend", "backend", "docs"]
}

variable "herramientas" {
  description = "Herramientas del equipo"
  type        = list(string)
  default     = ["docker", "terraform", "kubectl"]
}

variable "variables_entorno" {
  description = "Variables de entorno"
  type        = map(string)
  default = {
    ENV  = "development"
    PORT = "3000"
    LOG  = "debug"
  }
}

Parte 2

Recursos y outputs

2 Actualizar main.tf con for_each:

resource "local_file" "proyectos" {
  for_each = toset(var.proyectos)
  content  = "# Proyecto: ${each.key}\nEntorno: ${var.nombre_entorno}"
  filename = "${path.module}/dev/${each.key}/README.md"
}

resource "local_file" "env_file" {
  content  = join("\n", [for k, v in var.variables_entorno : "${k}=${v}"])
  filename = "${path.module}/dev/.env"
}

3 Crear outputs.tf:

output "proyectos_creados" {
  value = [for p in var.proyectos : p]
}

output "entorno" {
  value = var.nombre_entorno
}

4 Aplicar con valores por defecto:

terraform init && terraform apply

Parte 2

Personalizar con tfvars

5 Crear terraform.tfvars para personalizar:

nombre_entorno = "equipo-devops"

proyectos = ["api", "web", "worker", "monitoring"]

variables_entorno = {
  ENV      = "staging"
  PORT     = "8080"
  LOG      = "info"
  DB_HOST  = "localhost"
}

6 Destruir y re-aplicar:

terraform destroy
terraform apply
# Ahora se crean 4 proyectos en vez de 3
Clave

El mismo codigo Terraform genera infraestructura diferente segun las variables. En produccion, cada entorno (dev, staging, prod) tiene su propio .tfvars.

Parte 3

Definir contenedores

3 Crear main.tf con la red y contenedores:

resource "docker_network" "quizbattle" {
  name = "quizbattle-net"
}

resource "docker_container" "redis" {
  name  = "qb-redis"
  image = "redis:alpine"
  networks_advanced { name = docker_network.quizbattle.name }
}

resource "docker_container" "db" {
  name  = "qb-db"
  image = "postgres:15-alpine"
  env   = ["POSTGRES_USER=postgres", "POSTGRES_PASSWORD=postgres"]
  networks_advanced { name = docker_network.quizbattle.name }
}

resource "docker_container" "vote" {
  name  = "qb-vote"
  image = "dockersamples/examplevotingapp_vote"
  ports { internal = 80; external = var.vote_port }
  networks_advanced { name = docker_network.quizbattle.name }
}

resource "docker_container" "worker" {
  name  = "qb-worker"
  image = "dockersamples/examplevotingapp_worker"
  networks_advanced { name = docker_network.quizbattle.name }
}

resource "docker_container" "result" {
  name  = "qb-result"
  image = "dockersamples/examplevotingapp_result"
  ports { internal = 80; external = var.result_port }
  networks_advanced { name = docker_network.quizbattle.name }
}

output "vote_url" {

docker ps
# Debeis ver 5 contenedores corriendo

Abrid en el navegador: http://localhost:5000 (vote) y http://localhost:5001 (result)

Si algo falla
Error de provider
Comprobad que Docker Desktop esta corriendo
Puerto en uso
Cambiad vote_port o result_port en terraform.tfvars
Contenedor no arranca
docker logs qb-vote para ver errores
